## Challenges and Growth Areas

### Recurring Patterns (from deep story analysis)

1. **Architectural relocation inside a "visual-only" story introduced a behavioral regression.** Story 5.3 AC-3 moved `SessionStartSpinner` + Retry button from the always-visible input area into the scrollable `ChatMessageList` container. The auto-scroll `useEffect` (deps `[messages, isThinking]`) was not updated for the new location — `errorMessage` / `showRetry` / `showSpinner` are not in the dep array. Production-reachable: a `SESSION_TIMEOUT` while the user is scrolled up hides the Retry button below the fold. **Triply confirmed** by bug-hunt M1 + NFR-5.3-1 (Medium) + Wave-1 cross-reference (all verified at audit time). The only story-introduced Epic-5 regression with production-reachable impact.

2. **AC implementation gap hidden by a green test.** Story 5.3 AC-5 specifies two placeholder states: "Message…" (active) and "Message bmad-easy…" (branded). The implementation defaulted to "Message…" but the parent component never passes the branded variant on the new-conversation surface. The test asserted only the default, so the suite stayed green. Caught only by TEA validation (deferred per DP-5). Traceability weakness #2.

3. **`no-scrollbar` migration (Story 5.4 AC-7) missed one surface.** The two-pane artifact layout (line 93) received `no-scrollbar` correctly; the full-width list layout (line 123) did not. The test suite never renders the page without an `?id=` query param, so the gap is invisible. **Triply confirmed** by bug-hunt M2 + NFR-5.4-2 (Medium) + traceability weakness #1. The most actionable Wave-1 finding — fix is one CSS class + one test.

4. **Established pattern not followed by adjacent code.** Story 5.2's canonical depth-1 header pattern applies to all 4 depth-1 pages AND each one's `loading.tsx` companion (per `project-context.md:107`). Story 5.2's own review caught and patched `artifacts/loading.tsx`, but missed `conversations/[conversationId]/loading.tsx`. Bug-hunt L3 / NFR-5.2-3 surfaced the miss later. Pattern establishment needs an explicit all-files map at story-creation time and a completion audit, not case-by-case review.

5. **Pre-existing patterns surfaced when the dev touched their files.** Epic 5 modified files that contained pre-existing NFR-relevant gaps; touching the file surfaced the gap but the story scope (DP-5) refused the pull-in. The NFR epic report's "pre-existing findings surfaced" list:

   None introduced by Epic 5; all are recommended quick-win bundle candidates (see Action Items).

## Key Insights and Learnings

1. **A structural relocation inside a "visual-only" story is not a visual-only change.** Story 5.3 AC-3 moved spinner + Retry button across the boundary between the always-visible input area and the scrollable `ChatMessageList`. The auto-scroll effect's dependency array was tuned for the old location and not updated. **Lesson:** any AC that changes which element owns the rendering of state-conditional UI (spinner, retry, error) must trigger a fresh dependency-array audit. `react-hooks/exhaustive-deps` ESLint rule was on `warn`; elevating to `error` would have caught M1.

2. **Test coverage of an AC's positive case can hide its implementation gap.** Story 5.3 AC-5's branded placeholder test passed because it asserted only the default placeholder, not the wired-up branded variant. **Lesson:** every AC that specifies multiple states should test each state against the implementation's real source of truth (the parent component wiring the prop, not the component's own default). The TEA validation step caught this — a strong argument for running TEA validation as a default story step, not an optional one.

3. **Adjacent-code drift is invisible until adversarial review.** Story 5.2 established the canonical header pattern, updated both page.tsx and `artifacts/loading.tsx`, and missed `conversations/[conversationId]/loading.tsx`. Bug-hunt L3 found it later. **Lesson:** pattern establishment should map ALL files matching the pattern at story-creation time and audit the completion list explicitly. The current case-by-case review path misses anything sitting outside the immediate diff.

4. **Within-epic deferral handoff worked; cross-epic deferral still relies on manual triage.** Story 5.3 → Story 5.4 Save-button handoff worked because both stories were in the same epic and one dev did both. Cross-epic deferrals (per the SDK fidelity retro Action Item 1 — "open-concerns registry") need a tracking-key reference in `sprint-status.yaml` to survive; the scan script does not exist yet. **Lesson:** story-level deferral is fine for in-epic handoff; cross-epic deferral needs an explicit tracking-key — and the gap remains open.

5. **Wave-1+2 closeout assessment at epic end catches what per-story review missed.** Bug-hunt found M1 (auto-scroll), M2 (no-scrollbar full-width pane), M3 (parseFrontmatter quotes). Fidelity audit found the RepositoryUrlForm mock-shape drift. NFR audit aggregated per-story NFR findings and de-duplicated. Traceability exposed the 5.4-AC7 gap as triply confirmed. None of the per-story reviews had the cross-cutting vantage point needed to surface these. **Lesson:** invoke the fidelity auditor + bug-hunt + NFR audit + traceability at closeout as the standing minimum bar.

6. **Tailwind config guardrails are valuable AND safe when staged carefully.** No existing non-design-system utility was in use; the pre-switch grep + production build verified the switch was a guardrail, not a regression. The one missed key (`borderRadius.DEFAULT`) was caught by story review. **Lesson:** every config-level constraint change needs (a) a pre-switch grep, (b) a build-time test (the production build itself), (c) a regression test proving the constraint blocks the disallowed case.

7. **Deferral discipline (DP-5) was applied with rigor, but it requires an audit-able trail.** Every story has a "Deferred Work Pulled Into This Story" section listing each pulled-in finding's rationale and each not-pulled-in finding's reason. The Wave-1+2 audit then verified each "deferred" item remained sound. **Lesson:** DP-5 deferred findings must be checkable against an epic-closeout audit; the trail is what makes the discipline real.
