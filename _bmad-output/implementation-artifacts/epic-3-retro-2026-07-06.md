## Challenges and Growth Areas

### Recurring Patterns (from deep story analysis)

1. **`as SDKMessage` type-assertion bypasses — the SDK contract fidelity gap.** The most significant finding of the epic. `processAssistantMessage` cast `SDKMessage` to a hand-rolled shape, silently dropping image blocks and non-text content. 3 production bugs shipped behind 251 green tests because ts-jest's transpile-only mode silently dropped 11 type errors. Root cause: the pipeline had no contract-aware test instrument.

2. **Pre-existing deferred items compound across stories — 17+ entries.** Story 3-2's review found 13 pre-existing issues from Story 3-1 code. Story 3-3's review found the same `provisionQueue.release` in finally pattern from 3-1. Story 3-7's review found 4 pre-existing high/medium issues from 3-4. The deferral pattern from Epic 2 (sync error swallowing deferred 4x) continued: the same `provisionQueue.release` issue was deferred in 3-2, 3-3, and 3-7.

3. **`Date.now()` ID collisions — recurred across 3 stories.** `msg-${Date.now()}`, `error-${Date.now()}`, `tc-${Date.now()}` — collisions when two SSE events land in the same millisecond, causing duplicate React keys. First noted in 3-3, recurred in 3-4, partially fixed in 5-5 (tool-call segments use `safeUUID()`), but 4 other patterns persist.

4. **`onModuleDestroy` doesn't await in-flight promises — recurred across 3 stories.** First noted in 3-4 (classifier promises), recurred in 3-6 (working-tree promises), and 3-7 (classifier promises). Promises continue running after destroy, may emit to torn-down `SessionEventsService`.

5. **Circuit breaker race conditions — recurred across 2 stories.** `handleCircuitBreaker` emits `RUN_ERROR` but loop may still process a resolved result, emitting `TEXT_MESSAGE_END`/`TOOL_CALL_END`/`TOOL_CALL_RESULT` after the error. First noted in 3-4, recurred in 3-7.

## Key Insights and Learnings

1. **A contract test is only as good as the type enforcement behind it.** The SDK fidelity gap was not a test coverage problem — 251 tests passed. It was a type-enforcement problem: `as SDKMessage` bypassed the type system, and ts-jest's `diagnostics: false` (transpile-only) silently dropped the 11 type errors that would have caught it. The fix was adding `yarn nx run-many --target=typecheck --all` as a CI gate, not adding more tests.

2. **An external, contract-aware reviewer is necessary.** The `bmad-agent-fidelity-auditor` skill was the only instrument that could have caught this class of bug — the internal pipeline had no vantage point from which to see the contract drift. This validated the fidelity auditor as a load-bearing quality instrument, not just a one-off.

3. **The 2-strike escalation rule worked.** Epic 2 retro introduced it; Epic 3 applied it. Deferred items from 3-1/3-3/3-4 that blocked 3-9/3-11/3-12 were resolved as prerequisites, not deferred again. The rule prevented the Epic 2 pattern (sync error deferred 4x) from recurring at the same severity.

4. **Pre-existing issues dominate review findings in complex epics.** 13 of 3-2's findings were pre-existing from 3-1. 4 of 3-7's high/medium findings were pre-existing from 3-4. As epics grow in complexity, the ratio of new vs. pre-existing findings shifts toward pre-existing — the review becomes a debt-discovery instrument more than a new-bug-finding instrument.

5. **Fire-and-forget background pipelines need explicit lifecycle management.** The `onModuleDestroy` pattern (not awaiting in-flight promises) recurred across 3 stories because fire-and-forget work was added without a corresponding cleanup path. The pattern was eventually fixed in 3-12's drain sequence, but the recurrence shows that fire-and-forget is a code smell that needs explicit lifecycle annotation.

## Significant Discoveries and Epic Update Recommendations

### Significant Change Identified

The SDK contract fidelity gap exposed a structural blind spot in the pipeline: **type-assertion bypasses (`as Type`) are invisible to ts-jest's transpile-only mode, and no contract-aware test instrument existed to catch the drift.**

**Evidence:**
1. 3 production bugs shipped behind 251 green tests
2. `as SDKMessage` bypassed the type system; ts-jest's `diagnostics: false` silently dropped 11 type errors
3. The fidelity auditor (external, contract-aware) was the only instrument that caught it
4. The fix added a CI typecheck gate (`yarn nx run-many --target=typecheck --all`) — not just more tests

**Impact on Epic 4:**
- The typecheck gate is now a prerequisite for all stories
- The fidelity auditor is now a standing practice at epic closeout
- `as Type` assertions are treated as code smells requiring justification

**Epic update required: No.** The discovery was resolved within Epic 3 (fix + CI gate + re-audit). The lessons feed forward into Epic 4+ process.
