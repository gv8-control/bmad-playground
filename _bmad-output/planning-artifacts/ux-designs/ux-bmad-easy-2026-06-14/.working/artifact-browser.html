Problem Statement

Non-dev agile team members — PMs, BAs, Delivery Leads — cannot access BMAD methodology skills today. BMAD runs in Claude Code, which requires terminal access and developer context. This excludes the primary domain-knowledge holders from the workflow.

Key User Journeys

UJ-1 — Repository connection

A PM receives an invite link, connects a GitHub repository using a fine-grained PAT, and reaches the Project Map in under 3 minutes with no developer assistance.

UJ-2 — Skill Session and Commit Pill

A PM launches a /bmad-prd session, collaborates with the BMAD Agent over multiple turns, and sees the committed artifact appear inline as a Commit Pill at the moment of commit.
