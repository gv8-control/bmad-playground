### D-2: Color palette — indigo primary, ember accent
**Decided:** 2026-06-14
**By:** UX [ASSUMPTION — pending user review]
Primary: `#2C42DB` (vivid indigo-blue). Accent: `#EA5C1E` (warm ember orange). All other color tokens inherit shadcn defaults. Rationale: indigo signals structure and expertise without corporate-navy conservatism or startup-purple cliché; ember is the "live/active/committed" signal, reserved for moments of consequence (Commit Pills, active session indicators, slash-command prefix). Avoids gradient fills and decorative color.

### D-3: Navigation model — page-based, no in-app tab UI
**Decided:** 2026-06-14
**By:** PRD §8
Page-based navigation with stable URLs. Project Map is the root. Session and Artifact Browser are one level down with a breadcrumb. Concurrent sessions are browser tabs, not an in-app tab bar.

### D-4: Five primary surfaces
**Decided:** 2026-06-14
**By:** PRD §4
Auth → Onboarding → Project Map → Skill Session → Artifact Browser. Paywall is a gate state, not a distinct surface in the IA.

### D-5: Typography — Geist Sans throughout, Geist Mono for dev-adjacent content
**Decided:** 2026-06-14
**By:** UX [ASSUMPTION — pending user review]
No display font override. Geist Sans for all copy, labels, and navigation. Geist Mono restricted to: Skill names, slash commands, commit hashes, file paths, git-origin strings. Rationale: the product spans non-dev PMs (no terminal intimidation) and developer champions (authenticity where it counts); mono appears only where the content is inherently technical.

### D-6: Commit Pill as a distinct inline chat component
**Decided:** 2026-06-14
**By:** PRD §4.3 FR-12
Commit Pill is an inline chat-stream element positioned at the commit event, not a toast or sidebar notification. Styled as pill shape: muted background, ember-colored border, Geist Mono artifact name, ember "View →" link.

### D-7: Desktop / laptop as primary form-factor; mobile not targeted for MVP
**Decided:** 2026-06-14
**By:** PRD §8
Web only. Responsive behavior defined for `md` (768px) and `lg` (1024px) breakpoints. Mobile-phone viewports adapt but are not designed for.

### D-8: Ember accent is the sole "live" signal; never decorative
**Decided:** 2026-06-14
**By:** UX
Ember (`{colors.accent}`) appears only on: Commit Pills, active session pulse dots, the slash-command prefix in the Skill Launch Field, and the Working Tree State Indicator dirty state. It is not used for hover states, icons, section dividers, or decorative chrome. This restriction is what gives ember its meaning — when a user sees it, something has happened or is happening.

## Update: 2026-06-14 — PRD reconciliation (FR-18, FR-19, FR-20)

### D-9: Working tree state indicator uses ember for dirty state
**Decided:** 2026-06-14
**By:** UX (per PRD FR-18 + D-8)
The `● Unsaved changes` dirty state uses `{colors.accent}` (ember). Consistent with D-8: the indicator is live state — something is happening (uncommitted changes exist). Clean state uses muted foreground or is hidden.

### D-10: Platform-checkpoint Commit Pill uses dashed ember border to distinguish from agent-initiated
**Decided:** 2026-06-14
**By:** UX (per PRD FR-19, FR-12)
Agent-initiated Commit Pills use a solid `1.5px {colors.accent}` border. Platform-checkpoint Pills use a dashed `1.5px {colors.accent}` border. Both remain pill-shaped with muted background. Checkpoint Pills show a "Platform checkpoint" source badge + ISO timestamp; they do not show artifact type, artifact name, or "View →" (a checkpoint saves working state, not a complete artifact).

### D-11: FR-20 idle-timeout warning is an inline chat system message, not a Toast
**Decided:** 2026-06-14
**By:** PRD (FR-20 update) — overrides initial UX draft
Initial UX draft used a shadcn `Toast`. PRD (FR-20) specifies an inline system message in the chat timeline with an inline "Save now" action. Inline is correct: the warning is actionable and contextually tied to the session state visible in the chat; a dismissible toast in the corner is too easy to miss and doesn't pair naturally with a "Save now" CTA. Condition: only shown when working tree is dirty.

### D-13: Ember accent on small text — acknowledged borderline, deepen only if audit requires
**Decided:** 2026-06-14
**By:** UX (rubric MEDIUM finding — token completeness / contrast table)
The Working Tree Indicator's `● Unsaved changes` label renders ember (#EA5C1E) at 12px bold. Computed contrast against white: ~3.3:1 — fails WCAG AA for normal text (needs 4.5:1). The element is interactive (cursor-pointer) and the prefix dot is the primary signal. If a formal accessibility audit requires it, deepen that label only toward `#C44810` without altering other ember uses. Documented in DESIGN.md Colors contrast table.
