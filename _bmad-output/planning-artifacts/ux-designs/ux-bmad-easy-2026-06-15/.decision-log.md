## Design Decisions

### Visual direction
- **Decision:** "Feels like AI chat apps (Claude.ai, Perplexity) but with more specific tools. Keep it clean, not an IDE."
- **Source:** Marius, session 2026-06-15

### Color palette
- **Decision:** Custom palette drafted for this product. Dark violet-tinted near-black backgrounds (`#0D0D11`), violet accent (`#7B6EE8`), semantic colors: positive green (`#3ECF8E`), caution amber (`#F2A944`), negative red (`#F06B6B`).
- **Source:** Drafted in session per user direction

### Mode
- **Decision:** Dark mode only for MVP.
- **Source:** Marius, session 2026-06-15
- **[ASSUMPTION]:** Light mode is post-MVP

### Implementation stack
- **Decision:** React + Tailwind CSS. No named component library.
- **Source:** Marius, session 2026-06-15

### Form factor
- **Decision:** Desktop-first web app. Tablet supported with collapsed nav (off-canvas drawer). Mobile out of scope for MVP.
- **Source:** PRD §8 ("Web application, any modern browser. No mobile native app.")
- **[ASSUMPTION]:** Minimum viewport 1024px; tablet 768px+

### Side navigation
- **Decision:** Always visible in app shell at 240px fixed. Contains: last 5 conversations, New Conversation button, separator, Project Map, Artifact Browser, Settings (user avatar). No icons other than avatar.
- **Source:** Marius, 2026-06-15 (navigation notes, since merged into EXPERIENCE.md)

### Artifact Browser ordering
- **Decision:** Last-modified date descending (most recently committed artifact at top). Confirmed by Marius 2026-06-15.

### Conversation URL model
- **Decision:** New Conversation has no URL until first message sent. On send, transitions to `/conversations/:id`.
- **Source:** PRD FR-9

### Tool Pill expand behavior
- **Decision:** Clicking a Tool Pill toggles inline expansion to show raw tool input/output. Not modal.
- **Source:** Derived from PRD FR-12 (all tool calls visible)

### Navigation transitions
- **Decision:** No page transition animations in MVP.
- **[ASSUMPTION]:** If animations added post-MVP, they must respect `prefers-reduced-motion`.

### Tablet navigation
- **Decision:** Collapsed side nav as off-canvas drawer, triggered by hamburger in app header.
- **[ASSUMPTION]:** Not specified in PRD; derived from standard responsive pattern for fixed sidebars.

### Sign-in vs sign-up
- **Decision:** There is no separate sign-up screen. GitHub OAuth is the sole identity provider; account creation is transparent on first sign-in. `/sign-up` is removed. `/sign-in` serves both new and returning users.
- **Source:** Marius, 2026-06-15
- **Impact:** Pre-App Shell IA table reduced to Sign In + Onboarding. State pattern section renamed and "Form validation error" row removed (irrelevant for an OAuth-only flow).

### Onboarding model (2026-07-02 update)
- **Decision:** Onboarding is single-field (Repository URL only). No Access Token / PAT input. The GitHub OAuth token obtained at sign-in is reused to check write access to the repository. Validation failure now names the specific cause (insufficient permission, inaccessible repository, or GitHub org OAuth App restriction) rather than a generic error.
- **Source:** `architecture.md` DL-7 correction; `epics.md` UX-DR14 / Story 1.3.
- **Impact:** EXPERIENCE.md Flow 1 and Onboarding Flow States rewritten to drop the two-field PAT model (previously stale since ~2026-06-17, flagged in `implementation-readiness-report-2026-07-02.md`). `.working/key-onboarding.html` mock updated to match — access-token field and its "How to generate an access token" link removed.

### Epic 3 readiness audit (2026-07-04)
- **Context:** Ran in Update mode, autonomously, per Marius's instruction to prepare for Epic 3 without stopping to ask, governed by `_bmad-output/decision-policy.md`. Cross-checked every Epic 3 story's acceptance criteria (`epics.md`) against the finalized spine pair. Confirmed the 2026-07-02 finalization (concurrent-limit/seat-limit states, mid-Conversation credential banner, Access Notice, Artifact Card patterns, signin mock) already covers the bulk of Epic 3 — no re-litigation needed there. Also confirmed the intervening `epics.md` edit (commit `3260c58`, splitting old Story 3.5 into 3.5/3.11/3.12) and the one-line `architecture.md` edit (commit `1ec9f32`) introduced no new user-visible behavior, so neither required a spine update.
- Found three genuine coverage gaps — testable ACs with no corresponding UX spec, which would otherwise leave a downstream builder inventing the behavior:
  1. **Story 3.1/3.5 — session-start timeout.** AC requires "a retry affordance rather than an indefinitely spinning 'Starting session…' state" when `SESSION_READY` never arrives within a client-side timeout. No such state existed anywhere in EXPERIENCE.md.
  2. **Story 3.6 — working-tree disclosure.** AC requires help text "reachable" that is explicitly "distinct from triggering the save popover." The only interaction previously specified for the dirty indicator was click-to-save — there was no separate way to reach the disclosure copy.
  3. **Story 3.4 — circuit-breaker termination.** AC requires the backend to terminate the agent process and emit "an error event to the user" when `sandbox-agent` crashes or stalls — distinct from an ordinary failed-tool-call (already covered by the existing error-state Tool Pill pattern, which explicitly keeps the turn alive). No state existed for a whole-turn abort.
- **Decision (DP-3):** For all three gaps, several reversible, functionally-equivalent designs existed (e.g., full-page-reload retry vs. soft retry; hover-only tooltip vs. a separate info affordance; folding the circuit-breaker case into the existing Tool Pill pattern vs. giving it its own system message). Picked the simplest option consistent with patterns already established elsewhere in the spine:
  - Session-start timeout reuses the same "error text + single action button" shape already used for Conversation history load failure and Artifact load failure (`Couldn't ___. [Retry]`), rather than inventing a new visual pattern. Applies identically to first-open and to a stalled reconnect, so there's one pattern, not two.
  - The working-tree disclosure is a separate small `ⓘ` info affordance beside the indicator label (own Tab stop, own `aria-label`) rather than a hover-only tooltip on the label — a hover-only tooltip would be unreachable by keyboard and would conflict with Enter/Space already being bound to the Save-confirmation action on that same element (Accessibility Floor's keyboard-navigability requirement forecloses the simpler-looking hover-only option).
  - Circuit-breaker termination gets its own system message row in Conversation Surface States, kept deliberately distinct from the Tool Pill error pattern — folding it into a Tool Pill would misrepresent a whole-turn abort as a single failed call, and would contradict the existing Tool Pill spec's implicit "the turn continues" framing.
- **Changes:** EXPERIENCE.md — new "Session start timeout" paragraph under New Conversation; a numbered step added to Conversation Loading (Returning); one bullet added to Working Tree Indicator; new "Agent process terminated (circuit breaker)" row in Conversation Surface States plus a clarifying sentence in Agent Processing States; two lines added to Accessibility Floor (keyboard nav for the info affordance, `role="status"` for the new system message). DESIGN.md — two new `working-tree-indicator` frontmatter tokens (`info-icon-color`, `info-icon-color-hover`); Working Tree Indicator component prose updated for both states. Both files' `updated` set to 2026-07-04; `status` remains `final`.

### Aesthetics review fixes (2026-07-08)
- **Context:** Ran in Update mode, autonomously, per Marius's instruction to "handle all findings" from the aesthetics validation review (`review-aesthetics.md`, 2026-07-08). The review examined colors, margins/spacing, and consistency across both spines and all 7 key-screen mockups — a lens no prior review had covered. 3 critical, 6 high, 10 medium, 6 low findings.
- **Decision (DP-3):** For each finding, picked the simplest fix consistent with patterns already established in the spine. Key design decisions:
  1. **Broken `{colors.warning}` / `{colors.warning-bg}` tokens (critical):** Replaced with `{colors.caution}` / `{colors.caution-bg}` — the closest existing semantic ("something needs attention"). Adding a distinct `warning` token would create a fourth semantic color with no clear distinction from `caution`; the Access Notice's meaning ("rate-limited / org-restricted / insufficient-permission") maps cleanly to caution ("needs attention, not an error") for the first two codes and `negative` for `INSUFFICIENT_PERMISSION`.
  2. **`text-3` WCAG AA failure (critical):** Chose the hybrid approach — keep `text-3` at `#56556A` for WCAG-exempt contexts (placeholders, disabled text) and move permanent informational text (timestamps, "All saved" indicator, last-modified dates) to `text-2` (`#8D8CA0`, 5.08:1 on surface). Rejected the alternative (lightening `text-3` to ~`#85849A`) because on these very dark surfaces, 4.5:1 compliance compresses the three-level text hierarchy to near-indistinguishable — `text-3` at 4.57:1 is only 0.03 luminance away from `text-2` at 5.08:1. The hybrid preserves the visual hierarchy while remaining compliant.
  3. **`access-notice` missing frontmatter (critical):** Added `access-notice:` entry to the `components:` frontmatter with resolved caution/caution-bg token references, matching the `credential-error-banner` pattern.
  4. **`border-subtle` equals `surface-raised` (high):** Changed from `#1E1E26` to `#232330` — a value between `surface` (`#16161C`) and `border` (`#2B2B38`), visible on both `surface` and `surface-raised`.
  5. **4px grid violations (high):** Snapped all off-grid values in spine frontmatter to nearest 4px multiple. Standardized all pill/badge padding (tool-pill, semantic-pill, status-badge) to `4px 8px` — previously `2px 8px` / `4px 10px` / `2px 8px`. Fixed credential-error-banner (`10px` → `12px`), scroll-to-bottom-button (`6px` → `8px`), stop-button (`6px` → `8px`). Mockup off-grid values also snapped.
  6. **Sign-in button color (high):** Changed mock from `text-1` background to `accent` background, matching the onboarding button and the spine's Do's rule.
  7. **Cross-mock consistency (high):** Fixed conversation mock: nav-item `display: flex` → `block`, textarea color `text-3` → `text-1`, class name `.send-btn` → `.send-button`, chat-input bottom padding `20px` → `24px`.
  8. **Spacing scale tokenization (medium):** Added `'1'` through `'16'` scale tokens to the `spacing` frontmatter, matching the Tailwind scale.
  9. **Content-width system (medium):** Documented per-surface max-widths in DESIGN.md §Layout & Spacing as a table with rationale.
  10. **Floating-element shadow exception (medium):** Documented in §Elevation & Depth — shadows allowed for floating overlays (slash picker, popovers), never for layout surfaces.
  11. **Scrollbar styling (medium):** Documented the hidden-scrollbar decision in DESIGN.md §Layout & Spacing.
  12. **Wordmark treatments (medium):** Documented both pre-app-shell (full lockup) and app-shell (compact text mark) presentations in §Brand & Style.
  13. **Disabled/loading button states (medium):** Documented in Do's — `text-3` background with `text-2` text for disabled; `text-3` background with accent-fg spinner for loading.
  14. **Browser-chrome furniture (low):** Noted in Do's that mock browser-chrome elements are outside the design system scope.
  15. **Focus rings (medium):** Standardized spine prose; mock fixes applied.
  16. **Dead CSS, spinner standardization, ad-hoc colors (low):** Fixed in mockups.
- **Changes:** DESIGN.md — frontmatter: `border-subtle` value, `text-3` comment, `spacing` scale tokens, `working-tree-indicator.clean-color` text-3→text-2, pill/badge padding standardization (tool-pill, semantic-pill, status-badge ×2), scroll-to-bottom-button/stop-button padding, credential-error-banner padding, new `access-notice` component entry; prose: Access Notice tokens fixed, Colors table border-subtle value, contrast targets + text-3 restriction note, Typography timestamp text-3→text-2, Layout & Spacing content-width system + scrollbar note, Elevation floating-element exception, Brand & Style wordmark treatments, Messages timestamps text-3→text-2, Working Tree Indicator clean-color text-3→text-2, Do's disabled/loading/browser-chrome notes. EXPERIENCE.md — Access Notice tokens fixed, text-3→text-2 for clean-state indicator and last-modified timestamp, mockup viewport note in Foundation. Both files' `updated` set to 2026-07-08; `status` remains `final`. All 7 mockups updated (see below).
