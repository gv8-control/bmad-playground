Pattern 3
Blocking content message
Replaces the content area (chat area or content pane). Message + optional action button. Input is hidden or disabled. Decision (2026-07-16): carries {colors.negative} — negative-bg background, negative left border, negative icon. Six variants across two surfaces.
Design decision needed: No DESIGN.md component spec for this pattern. The spine references "same visual treatment" but never defines it. The renders below use a neutral centered treatment — is that right, or should load failures carry negative color?

Pattern 5
Access Notice
Inline notice rendered in the message stream directly below an error-state Tool Pill. Scoped to a single failing tool call — not full-width, not pinned. Dismissible. Never offers re-auth (a 403 is not a credential failure). Component spec: {components.access-notice}. Three variants by ACCESS_DENIED code.

Pattern 6
Error-state Tool Pill
Inline chip in the agent message stream at the position a tool call failed. Clicking expands to show the raw error output. Decision (2026-07-16): the error-state variant stays neutral — same styling as the base Tool Pill ({components.tool-pill}). The error signal is carried by the Access Notice or Credential Error Banner below/above it. Status text ("✕ failed") differentiates.
Design decision needed: No DESIGN.md spec for the error-state variant. The render below uses negative border + negative text + negative-bg — is that right, or should it stay neutral and let the Access Notice carry the error signal?
