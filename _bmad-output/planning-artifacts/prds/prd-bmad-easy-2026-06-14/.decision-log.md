## DL-9 — Semantic Pill emitted only on confirmed commit success
**Date:** 2026-06-15
**Decision:** The "Progress saved" Semantic Pill is emitted only after confirmed commit success — not on commit initiation. A failed `git commit` produces an error-state Tool Pill; FR-14's working tree state indicator remains dirty.
**Rationale:** Prevents false "Progress saved" signal to users when commits fail. A first-session user who sees "Progress saved" for an artifact that was not committed will not return.
**Affected sections:** §4.3 FR-12.
**Status:** Active.

## DL-7 — Repository git authentication: OAuth App + `repo` scope (replaces PAT)
**Date:** 2026-06-15
**Decision:** Git transport credential for Repository operations is the GitHub OAuth access token obtained at sign-in with `repo` scope. The original PRD used a fine-grained PAT (FR-1, A-6). The PM explicitly chose OAuth App + `repo` scope over the recommended GitHub App model.
**Rationale stated by PM:** Simplicity; single-step authorization; avoids implementation complexity of GitHub App token management; avoids two-step onboarding (install + OAuth).
**Known trade-offs accepted:**
- OAuth token with `repo` scope grants access to ALL repositories the user can access, not only the connected one. This is broader than needed and cannot be narrowed within the OAuth App model.
- Org-level GitHub OAuth App restrictions (if enabled by the org owner) can block write access to organization repositories for non-owner members, causing silent failures. This is a platform limitation, not a user error.
- Long-lived credential stored by the platform (until revoked by the user); no automatic expiry.
**Conflict with research recommendation:** Research recommended GitHub App model (fine-grained, short-lived tokens, repo-scoped). PM overrides this recommendation. GitHub App remains the documented post-MVP path.
**Affected sections:** UJ-1, §4.1, FR-1, FR-3, FR-4, §4.5 description, Glossary (PAT entry removed), §6.1, §6.2, NFR-S2, NFR-S3, NFR-S5, A-6, §8 (new org OAuth restriction constraint added).
**Status:** Active.
